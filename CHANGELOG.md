# Changelog

## 0.7.0 - 2026-07-16

- Renames the public project to AskUp: the decision system that teaches an agent to manage up.
- Publishes the complete Public Alpha package: examples, evaluation protocol, validation record, approval contracts, security guidance, and contributor guidance.

## 0.5.0 - 2026-07-15

- Adds Light, Standard, and Strict operating modes with clear stop conditions.
- Adds risk tiers and an approval-ready Decision Card template.
- Tightens skill discovery around high-impact actions and approval-ready decisions.

## 0.4.0 - 2026-07-15

- Introduces the initial agent-builder framing for developers, operators, founders, and teams.
- Replaces the technical README with concrete decision-boundary scenarios, transparent current limits, and a public roadmap.
- Adds repository maintenance, contribution, and security policies.

## 0.3.0 - 2026-07-15

- Defines the project as a decision-policy layer and documents the enforcement boundary.
- Adds versioned DecisionRecord and ApprovalRequest contracts for future host adapters.
- Changes the repository license to Apache-2.0 to support enterprise integration and future adapter code.
- Adds staged release gates for evidence, enforcement, and preference governance.

## 0.2.0 - 2026-07-15

- Adds a four-part Question Gate that makes question selection inspectable.
- Adds reportable evaluation measures, a third example, and an initial blind-test record with limitations.
- Sharpens the public positioning around decision quality, evidence, and approval boundaries.

## 0.1.0 - 2026-07-15

- Initial public decision-policy draft.
